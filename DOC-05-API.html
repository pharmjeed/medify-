DOC-05 — عقد API الكامل

FastAPI · القاعدة /api/v1 · هذه القائمة حصرية: ممنوع إنشاء أي نقطة نهاية غير مذكورة هنا بعد الاعتماد.
الإصدار: 1.0 التاريخ: 2026-07-06 الحالة: معتمدة ✓ المالك: عبدالمجيد التركي

١ الاتفاقيات العامة

  • المصادقة: JWT Bearer قصير العمر (30 دقيقة) + Refresh Token (7 أيام، httpOnly cookie). الحمولة تتضمن user_id, facility_id, role — ومنها تُضبط جلسة RLS (DOC-04 §٧).
  • الصلاحيات: عمود «الدور» أدناه هو المصدر المبدئي؛ المصفوفة الكاملة في DOC-06.
  • الترقيم الصفحي: ?page=1&per_page=25 (الحد الأقصى 100)، والاستجابة تتضمن meta.total.
  • التزامن: عمليات التعديل على الملخص تتطلب If-Match: <etag> لمنع الكتابة فوق تعديل أحدث.
  • حصر المعدل: 429 مع Retry-After؛ حدود خاصة أشد لنقاط الـAI.

٥ بروتوكول WebSocket — التفريغ الفوري

المسار: WSS /ws/visits/{id}/transcribe?token= — يعمل فقط والزيارة في حالة recording.

// العميل → الخادم
{ "type": "audio_chunk", "seq": 41, "payload": "<base64 opus 250ms>" }
{ "type": "pause" } | { "type": "resume" } | { "type": "end" }

// الخادم → العميل
{ "type": "partial",  "seq": 41, "text": "يشتكي المريض من ..." }      // ≤ 2s (NFR-01)
{ "type": "final",    "segment_id": "s-17", "text": "...", "t0": 12.4, "t1": 15.1 }
{ "type": "status",   "state": "summarizing" }                        // بعد end
{ "type": "error",    "code": "MDF-5031" }
  • انقطاع الشبكة (NFR-09): العميل يخزّن الأجزاء محلياً ويعيد الإرسال من آخر seq مؤكد عند إعادة الاتصال — الخادم يرد بـ{type:"resume_from", seq}.

٦ واجهة الرفع لنظام المستشفى (خارجية)

الرفع أحادي الاتجاه بصيغة FHIR متوافقة NPHIES إلى النقطة المهيأة في FR-302. الحزمة: Bundle يضم Encounter + Composition (SOAP) + Condition[] + MedicationRequest[] + Procedure[] بالرموز المعتمدة. تفاصيل الربط العميق ثنائي الاتجاه مؤجلة لـDOC-19.

مقفول: لا نقطة نهاية خارج هذه الوثيقة · كل نقاط الدكتور تمر بفلتر doctor_id فوق RLS · /approve هي المسار الوحيد الذي يُنشئ upload_job · رموز الأخطاء حصرياً من DOC-13.
