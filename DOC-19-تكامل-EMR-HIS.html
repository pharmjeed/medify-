DOC-19 — تكامل EMR/HIS العميق

خارطة الانتقال من الرفع أحادي الاتجاه (الإطلاق الأول) إلى التكامل ثنائي الاتجاه. لا يبدأ تنفيذها قبل اجتياز DOC-17.
الإصدار: 1.0التاريخ: 2026-07-06الحالة: بانتظار الاعتمادالمالك: عبدالمجيد التركي

١ الوضع الحالي والهدف

الإطلاق الأول (مقفول)المرحلة الثانية (هذه الوثيقة)
ملف المريضلقطة تُجلب عند بدء الزيارة عبر وجهة الربط المهيأة (FR-601)مزامنة حية: مشكلات، أدوية، حساسيات، نتائج مختبر — عبر FHIR API للنظام المضيف
الرفعأحادي الاتجاه: Bundle بعد الاعتماد (DOC-05 §٦)كتابة مباشرة في السجل + تأكيد معرفات مقابلة (Encounter ID المضيف)
الجدولةلا شيء — الدكتور يبدأ الزيارة يدوياًقائمة مواعيد اليوم من النظام المضيف تفتح الزيارة بنقرة

٢ مسارات العمل

  1. مصفوفة الأنظمة المستهدفة: حصر أنظمة HIS الشائعة في المستشفيات الخاصة السعودية لدى عملاء التجربة أولاً — التكامل يُبنى لنظام العميل الفعلي الأول، لا افتراضياً (التحقق قبل الالتزام).
  2. اعتماد NPHIES كـIntegrator: استكمال متطلبات المطابقة: FHIR HL7 profiles سعودية، PKI وشهادات، بيئة NPHIES الاختبارية ثم الإنتاجية.
  3. خرائط الترميز لـNafais: مواءمة الداخلي مع ICD-10-AM / ACHI / SBS / SFDA GTIN بإصداراتها المرخصة — جدول coding_system_configs يتوسع بإصدارات مؤرخة.
  4. التوسع الحكومي: التجمعات الصحية تتطلب هذا التكامل مكتملاً — يسبق أي عرض عليها (DOC-02 §٥).

٣ المتطلبات المؤجلة المعرّفة مسبقاً

  • نطاق FR جديد يُفتح حينها: FR-900..FR-999 (خارج نطاق DOC-03 المقفول الحالي — يتطلب إصدار PRD جديداً معتمداً).
  • جداول جديدة متوقعة: ehr_connections, sync_jobs, external_id_mappings — تُعتمد بإصدار ERD لاحق.
  • شاشات جديدة: نطاق W-300+ يُفتح بإصدار DOC-10 لاحق.
بنود تحقق قبل التنفيذ: وثائق API الفعلية لنظام العميل الأول · تكلفة ومدة اعتماد NPHIES الفعلية · رسوم تراخيص أنظمة الترميز (IHACPA/CCHI).
مقفول: لا تنفيذ لأي بند هنا قبل اجتياز العيادة التجريبية · التكامل الأول يُبنى لعميل حقيقي واحد · فتح النطاقات الجديدة يتطلب إصدارات وثائق معتمدة لا قرارات جلسة.

اكتملت المنظومة: 19/19 وثيقة. باعتماد هذه الدفعة (DOC-07..19) تُقفل المرحلة الوثائقية ويُفتح S0 من خطة السبرنتات.

للاعتماد: «اعتمد» — أو اطلب تعديلات محددة على أي وثيقة.
