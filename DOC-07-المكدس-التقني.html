٢ قائمة المحظورات — ملزمة لكل جلسة تنفيذ

  • ممنوع أي إطار CSS خارجي (لا Tailwind ولا Bootstrap) — التنسيق من نظام التصميم DOC-11 فقط.
  • ممنوع TypeScript غير صارم — strict: true إلزامي ولا any غير مبرر.
  • ممنوع أي نقطة API خارج DOC-05، وأي جدول/عمود خارج DOC-04، وأي شاشة خارج نطاق DOC-10.
  • ممنوع خروج بيانات مرضى خارج المملكة — يشمل أي خدمة طرف ثالث بلا منطقة سعودية.
  • ممنوع تطبيق أي إرشاد AI تلقائياً دون فعل صريح من الدكتور (FR-704).
  • ممنوع إنشاء upload_job بغير مسار /approve — على مستوى القاعدة لا الكود فقط.
  • ممنوع تسجيل محتوى سريري نصي في اللوجات أو التحليلات (DOC-14).
  • ممنوع تبديل نموذج AI معتمد دون اختبار مقارن موثق على عينة DOC-17 نفسها.
  • ممنوع مكتبات خارجية في ملفات HTML التسليمية — ذاتية الاكتفاء دائماً.
  • ممنوع أسرار في الكود أو المستودع — أسرار البيئة عبر متغيرات مشفرة فقط.

٣ قواعد التسليم والنشر

  • المستودع: github.com/pharmjeed/mednote-ai — دفع Git بيد المؤسس حصراً (منهجية خامساً).
  • كل مخرج واجهات ثابتة يُسلَّم zip جاهزاً لـNetlify Drop وindex.html في الجذر دون مجلد زائد.
  • جلسات تنفيذ متوازية مسموحة على وحدات لا تتقاطع ملفاتها (واجهة الأدمن / واجهة الدكتور / خطوط المعالجة / الرفع).
  • هجرات Alembic مرقّمة تسلسلياً ولا تُعدَّل هجرة مدموجة.
