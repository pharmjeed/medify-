# Medify

منصة توثيق سريري عربية متعددة المنشآت، تحول الزيارة إلى مسودة SOAP وإرشادات قابلة للمراجعة، وتمنع تصدير السجل قبل الاعتماد البشري الصريح.

## المكدس

- Next.js 14 وTypeScript للواجهة العربية RTL.
- FastAPI وPython لواجهات التطبيق وخطوط المعالجة.
- PostgreSQL 16 للبيانات وRedis للجلسات والتحكم التشغيلي.
- Alembic للترحيلات وDocker Compose للنشر المعزول.

## الوظائف المنفذة

- صلاحيات مدير المنشأة والطبيب وعزل البيانات حسب المنشأة.
- إدارة العيادات والأطباء والمقاعد والقوالب.
- رحلة زيارة متتابعة: مريض، قالب، تسجيل، تفريغ، SOAP، إرشادات، مراجعة واعتماد.
- منع التعديل بعد الاعتماد وإنشاء بصمات للمحتوى المعتمد.
- حزمة FHIR Transaction بعد الاعتماد فقط.
- موافقات المرضى وطلبات حقوق أصحاب البيانات.
- جلسات HttpOnly، Argon2id، قفل محاولات الدخول، rate limiting، ورؤوس أمان.
- سجل تدقيق بسلسلة hash قابلة للتحقق.
- تشفير إعدادات التكامل، ترحيلات قاعدة بيانات، نسخ احتياطي واختبار استعادة.
- اختبارات صحة وأمان وعزل مستأجرين ورحلة كاملة.

## التشغيل

1. انسخ `.env.example` إلى `.env` وولّد أسرارًا قوية.
2. شغّل `docker compose -p medify up -d --build`.
3. افتح `http://localhost:3100` في التطوير فقط.
4. للإنتاج استخدم نطاق HTTPS واضبط `COOKIE_SECURE=true` و`ALLOWED_ORIGINS`.

تنفذ ترحيلات Alembic تلقائيًا قبل بدء API. فحص العملية على `/api/health` وفحص PostgreSQL وRedis على `/api/ready`. وثائق API متاحة خارج وضع الإنتاج فقط.

## العرض التجريبي

عند `DEMO_MODE=true` فقط:

- المدير: المنشأة `demo`، المستخدم `admin`، كلمة المرور `Admin123!`.
- الطبيب: المنشأة `demo`، المستخدم `doctor`، كلمة المرور `Doctor123!`.

لا تستخدم بيانات مرضى حقيقية في وضع العرض. في الإنتاج يكون التسجيل العام والمحاكاة معطلين، ولا يدعي النظام نجاح أي ربط خارجي دون إعداد واعتماد حقيقي.
