DOC-06 — مصفوفة الصلاحيات RBAC

دوران فقط: admin و doctor (قرار مقفول من DOC-01). المصفوفة حصرية وتُنفَّذ على ثلاث طبقات: API ثم RLS ثم قيود القاعدة.
الإصدار: 1.0 التاريخ: 2026-07-06 الحالة: معتمدة ✓ المالك: عبدالمجيد التركي

١ مبادئ التنفيذ

ثلاث طبقات دفاع: (1) حارس دور على كل نقطة API · (2) RLS بالعزل على facility_id وفلتر doctor_id · (3) قيود قاعدة البيانات (إلحاقية approvals، آلة حالات visits). فشل أي طبقة لا يكشف البيانات.
الفصل الجوهري: الأدمن يدير البنية ويرى التجميعات — ولا يقرأ أبداً المحتوى السريري النصي (تفريغ، ملخص، إرشادات). الدكتور يملك المحتوى السريري — لزياراته هو فقط — ولا يمس البنية.

الرموز: ✓ مسموح · ✗ ممنوع · ◐ مشروط (الشرط مذكور).

٤ قواعد قاطعة (تتقدم على أي تفسير)

  • لا يوجد دور «مدير نظام خارق» يتجاوز العزل — دعم Medify التقني نفسه لا يقرأ محتوى سريرياً؛ أي تدخل دعم يتم بجلسة موثقة بموافقة المنشأة ويُسجل في audit_logs.
  • الاعتماد (/approve) لا يُفوَّض: لا يعتمد دكتور عن دكتور، ولا الأدمن عن أحد.
  • كل استجابة 403 تستخدم رمز خطأ من DOC-13 دون كشف وجود المورد من عدمه (404 للموارد خارج نطاق الرؤية).
  • أي نقطة API جديدة مستقبلاً تُضاف أولاً هنا وفي DOC-05 قبل أي تنفيذ.
